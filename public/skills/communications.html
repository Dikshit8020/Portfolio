01 / Overview

Wired & Wireless Experience

Communication protocols are the nervous system of every embedded project. The robotic arm uses SPI to configure 6 TMC5160 stepper drivers. The Li-Fi system uses UART as the backbone of its optical data link. The EMI system uses SPI for ADC data transfer and USB/serial for RPi communication.

On the wireless side, I've built IoT sensor nodes with ESP32 BLE + Wi-Fi, experimented with LoRa for long-range agricultural monitoring, and used ZigBee mesh networks for multi-node deployments.

04 / Code Sample

SPI Daisy-Chain for TMC5160

// tmc5160_spi.c — SPI config for stepper driversembedded_c
/* Configure TMC5160 via SPI.
   6 drivers daisy-chained on one SPI bus.
   Each gets unique CS line for register access. */

void tmc_write_reg(uint8_t axis, uint8_t reg,
                    uint32_t val) {
    uint8_t tx[5] = {
        reg | 0x80,  // Write flag
        (val >> 24) & 0xFF,
        (val >> 16) & 0xFF,
        (val >> 8)  & 0xFF,
         val        & 0xFF
    };
    HAL_GPIO_WritePin(cs_ports[axis],
                      cs_pins[axis], GPIO_PIN_RESET);
    HAL_SPI_Transmit(&hspi1, tx, 5, 100);
    HAL_GPIO_WritePin(cs_ports[axis],
                      cs_pins[axis], GPIO_PIN_SET);
}

/* Example: Set motor current and StealthChop */
tmc_write_reg(0, IHOLD_IRUN, 0x00061F0A);
tmc_write_reg(0, GCONF,      0x00000004);
tmc_write_reg(0, TPWMTHRS,   0x000001F4);
