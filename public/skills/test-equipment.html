01 / Philosophy

Measure, Don't Assume

The most dangerous bugs in embedded systems are the ones firmware can't see. A UART line that looks perfect in code might have 400mV ringing on the oscilloscope. An SPI transfer that returns correct data 99.9% of the time might be violating setup time by 50ns — and you'll only catch it with a logic analyzer triggering on the CS edge.

My debugging philosophy: if it's intermittent, it's analog. Don't trust printf debugging for timing issues. Hook up the scope.

03 / Debug Stories

Real Problems I've Solved

Li-Fi Project · Month 3
400mV Power Rail Spike From Laser Driver
The NPN transistor switching the laser caused inductive kickback from the diode's bond wire inductance (~5nH). Visible only on the oscilloscope — firmware showed perfect UART data. Fixed with TVS diode + series resistor to slow the switching edge.
🔬 Oscilloscope + 10× probe
Robotic Arm · Week 12
TMC5160 SPI Register Corruption
StealthChop mode randomly disabling. Logic analyzer showed SPI MOSI had 200ns glitches during stepper motor switching (ground bounce). Added 100nF bypass caps on each TMC5160 VCC pin — fixed immediately.
🔬 Logic analyzer + protocol decode
Internship · Calibration Tool
IR Pyrometer Reading ±15°C Drift
Calibration firmware looked correct. Used oscilloscope to measure ADC reference voltage — it was drifting 30mV with ambient temperature. The LM385 voltage reference was improperly bypassed. Added 10µF + 100nF decoupling — drift reduced to ±0.5°C.
🔬 Oscilloscope + DC measurement
