01 / Context

ML for Hardware Engineers

I approach ML differently from a software engineer. My interest is embedding intelligence into hardware systems — not building web services. The EMI project is the perfect example: a 4-class CNN that classifies electromagnetic interference sources in real-time on a ₹5000 Raspberry Pi, replacing a ₹50L lab instrument.

The key constraint is always inference speed and memory. A model that takes 2 seconds is useless for a real-time EMI scanner. INT8 quantization via TFLite got our model from 140ms to 12ms — fast enough to classify every sweep cycle.

03 / Code Sample

CNN Architecture for EMI Classification

// emi_model.py — TensorFlow CNN architecturepython
import tensorflow as tf

model = tf.keras.Sequential([
    # Input: 1024-point FFT magnitude spectrum
    tf.keras.layers.Conv1D(32, 7,
        activation='relu',
        input_shape=(1024, 1)),
    tf.keras.layers.MaxPooling1D(4),
    tf.keras.layers.Conv1D(64, 5,
        activation='relu'),
    tf.keras.layers.MaxPooling1D(4),
    tf.keras.layers.Conv1D(64, 3,
        activation='relu'),
    tf.keras.layers.GlobalAveragePooling1D(),
    tf.keras.layers.Dense(64, activation='relu'),
    tf.keras.layers.Dropout(0.3),
    tf.keras.layers.Dense(4, activation='softmax')
])

model.compile(
    optimizer='adam',
    loss='categorical_crossentropy',
    metrics=['accuracy']
)
# Result: 94% test accuracy after 50 epochs
Key Optimization: INT8 Quantization
Post-training quantization via tf.lite.TFLiteConverter with representative dataset calibration reduced model size from 240KB to 65KB and inference time from 140ms to 12ms on RPi 4B — with only 0.8% accuracy loss.
