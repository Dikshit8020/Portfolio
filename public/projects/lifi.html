A fully custom bi-directional optical wireless communication link using visible light — no RF, no Wi-Fi. Built from scratch: laser TX driver, high-sensitivity photodiode RX chain, power-rail protection circuitry, and custom PCB. Achieved 1200 baud reliable data transfer over 5 metres.

01 / The Problem

Why Li-Fi?

Radio frequency spectrum is finite and increasingly congested. Hospitals, aircraft, and secure facilities restrict RF. Li-Fi — using light to carry data — offers a compelling alternative: immune to RF interference, inherently confined to a room, theoretically terahertz of available bandwidth.

The academic challenge: most Li-Fi research uses expensive avalanche photodiodes and lab-grade amplifiers. Our goal was a working system using off-the-shelf components under ₹2,000 total.

What Makes This Hard
A photodiode produces picoamperes of current from incident light. Amplifying that to a usable digital signal without amplifying noise requires careful analog design — the right transimpedance amplifier, minimal PCB stray capacitance, and clean power rails. Get any of these wrong and you amplify noise more than signal.

Why OPA388 for the TIA?
Zero-drift (0.25µV/°C offset) and rail-to-rail output — critical for a single-supply TIA where the output needs to swing cleanly between 0V and 3.3V. 10MHz GBW is more than sufficient for 1200 baud. The 1MΩ feedback resistor gives 1V/µA transimpedance gain — enough to swing 2V+ from a 2µA photodiode current change at 5 metres.
03 / The Journey

Month by Month

Month 1
Concept & First Breadboard Test
Tested basic concept: ESP32 blinking an LED, photodiode + LM358 detecting it. Worked at 10cm, completely failed at 1m. The LM358's slow slew rate and non-rail-to-rail output were the bottleneck.
❌ Range: 10cm max. LM358 too slow.
Month 2
Laser TX + OPA388 TIA Design
Switched from LED to 650nm laser — dramatically improves range due to collimation. Designed OPA388 TIA with 1MΩ feedback and 10pF compensation capacitor. 1m range achieved, but signal had noisy edges due to stray capacitance.
⚡ Range improved to 1m. Signal noisy.
Month 3
Power Rail Voltage Spike Problem
When the laser driver transistor switched, it caused a 400mV spike on the 5V rail — coupling into the TIA and corrupting the RX signal. Tried bulk capacitance (10µF + 100nF) — reduced to 200mV, not enough.
❌ Voltage spikes corrupting RX signal
Month 4
Inductive Kickback Root Cause & Fix
Used oscilloscope to identify the spike source: inductive kickback from the laser diode's bond wire inductance (~5nH) during NPN turn-off. Fixed with TVS diode (SMBJ5.0A) across laser + 10Ω series resistor. Spikes reduced to <20mV.
✓ Clean power rail. RX signal clean.
Month 5
PCB Design & Fabrication
Designed custom 2-layer PCB in KiCad. TIA input traces kept under 5mm. Analog and digital grounds split, joined at single star point. JLCPCB fabricated. First revision worked — rare for analog PCBs.
✓ PCB functional on first revision
Month 6
Bi-directional Operation & Testing
Full-duplex using separate TX/RX laser paths. BER at 1200 baud over 0.5m to 5.5m. System operates reliably up to 5m, BER <0.1% in indoor lighting conditions.
✓ 5m range · BER <0.1% · 1200 baud
04 / Technical Deep Dive

The Analog Design Details

// ESP32 — UART-to-laser transmissionembedded_c / arduino
// ESP32 transmits standard UART frames.
// The laser driver circuit (NPN + TVS + R) handles
// electrical-to-optical conversion transparently.

const int UART_TX_PIN  = 17;
const int UART_RX_PIN  = 16;
const int BAUD_RATE    = 1200;

void setup() {
    // Serial2 maps to the optical link
    Serial2.begin(BAUD_RATE, SERIAL_8N1,
                  UART_RX_PIN, UART_TX_PIN);
    Serial.begin(115200); // Debug USB serial
}

void loop() {
    // Echo received optical data to USB debug
    while (Serial2.available()) {
        char c = Serial2.read();
        Serial.print(c);
    }
    // Send test pattern every 500ms
    static uint32_t last = 0;
    if (millis() - last > 500) {
        Serial2.println("LIFI_OK");
        last = millis();
    }
}
Key Insight: Schmitt Trigger Hysteresis
The OPA350 comparator uses 100kΩ / 1MΩ positive feedback for ~200mV hysteresis. Without this, ambient light from fluorescent lighting (100Hz) caused false triggering. The hysteresis makes the output snap decisively between 0 and 3.3V.

08 / Learnings

What I'd Do Differently

  • Use an integrated TIA IC like OPT101. The OPA388 + external 10pF works but requires careful layout. An integrated solution would simplify dramatically.
  • Add a 650nm bandpass optical filter to the RX. Fluorescent and sunlight interference limited outdoor usability.
  • Implement Manchester encoding. Plain UART has long same-level runs. Manchester guarantees transitions every bit — self-clocking and DC-balanced.
  • Plan the ground plane split from schematic stage. Adding split analog/digital ground as an afterthought caused a crosstalk issue fixed with a bodge wire.
