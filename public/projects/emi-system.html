A portable, real-time electromagnetic interference measurement system built for DMRC (Delhi Metro Rail Corporation). Combines high-speed ADC sampling via NXP LPC4300, digital filtering through Xilinx CPLD, and a TensorFlow-based anomaly classifier running on Raspberry Pi 4B — all packaged in a field-deployable unit.

01 / The Problem

Why EMI Monitoring Matters

Delhi Metro's signalling systems, traction motors, and 25kV overhead lines generate broadband electromagnetic interference that can degrade nearby communication equipment, GPS receivers, and safety-critical control systems. Current EMI measurement requires ₹50L+ lab-grade spectrum analyzers — too expensive and bulky for field deployment.

DMRC needed a portable, sub-₹1L device that field engineers could carry on-site, measure EMI levels in real-time, and automatically classify the source (traction, signalling, HVAC, or external).

The Real Challenge
EMI signals span 100kHz to 1GHz. Sampling at Nyquist requires 2 GS/s — impossible for a portable device. Our approach: use the CPLD as a hardware frequency hopping controller, sweeping a mixer through bands while the LPC4300's 80 MS/s ADC captures 40MHz windows. Software-defined spectrum stitching reconstructs the full picture.

03 / The Journey

Development Timeline

Phase 1
Antenna Design & LNA Characterization
Tested 3 antenna types: loop, monopole, and biconical. Loop antenna gave best sensitivity in 1–100MHz range (most metro EMI). LNA board designed with AD8331 VGA for auto-gain control based on signal strength.
✓ -60dBm sensitivity achieved
Phase 2
CPLD Frequency Hopping Controller
Xilinx CoolRunner-II CPLD programmed in VHDL to control the SA612 mixer's local oscillator. 32 frequency bands, 10ms dwell per band, synchronised with LPC4300 ADC triggering. Total sweep time: 320ms for full 100kHz–1GHz coverage.
✓ 320ms full-spectrum sweep
Phase 3
LPC4300 High-Speed ADC Integration
NXP LPC4370's built-in 80MS/s ADC sampled IF output from mixer. DMA circular buffer streams data to SRAM. SPI bridge sends processed data blocks to RPi. Challenge: ground bounce on ADC inputs from digital switching — fixed with split ground planes.
⚡ Ground plane redesign needed
Phase 4
ML Model Training & Edge Deployment
Collected 2000+ EMI samples from metro environment (traction, signalling, HVAC, external). Trained CNN in TensorFlow (3-layer conv + dense), achieved 94% test accuracy. Converted to TFLite INT8 — inference time 12ms on RPi 4B. Fast enough for real-time classification.
✓ 94% accuracy · 12ms inference
04 / Technical Deep Dive

The ML Classifier Code

// emi_classifier.py — TFLite inference on RPi4python
import numpy as np
import tflite_runtime.interpreter as tflite

# Load quantized INT8 model for fast inference
interpreter = tflite.Interpreter(
    model_path="emi_classifier_int8.tflite")
interpreter.allocate_tensors()
input_details  = interpreter.get_input_details()
output_details = interpreter.get_output_details()

CLASSES = ["Traction", "Signalling",
           "HVAC", "External"]

def classify_emi(fft_magnitudes):
    """Classify EMI source from FFT spectrum.
    Returns (class_name, confidence)."""
    inp = np.expand_dims(
        fft_magnitudes.astype(np.float32), axis=0)
    interpreter.set_tensor(
        input_details[0]['index'], inp)
    interpreter.invoke()
    probs = interpreter.get_tensor(
        output_details[0]['index'])[0]
    idx = np.argmax(probs)
    return CLASSES[idx], float(probs[idx])

07 / Learnings

What I'd Do Differently

  • Use an SDR frontend (RTL-SDR) instead of discrete mixer. Our SA612 mixer + CPLD approach works but limits bandwidth. An RTL-SDR dongle provides 0.5–1.7GHz coverage with better linearity for ~₹2000.
  • Train the ML model with more diverse data. Our 2000-sample dataset was collected in one metro station. Adding data from multiple stations and environmental conditions would improve generalization.
  • Implement on-device model update via OTA. As new EMI sources appear, the model should adapt. Federated learning on RPi would allow updating without replacing the device.
